---
title: List Addresses
description: Query addresses with filters and pagination
---

<span class="http-method get">GET</span>
<span class="endpoint-path">/addresses</span>

List addresses with optional filters and pagination.

---

## Query Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `chain` | string | - | Filter by chain code |
| `external_id` | string | - | Filter by external ID |
| `limit` | integer | 50 | Results per page (max 100) |
| `offset` | integer | 0 | Pagination offset |

---

## Example Request

```bash
curl "https://api.yourdomain.com/v1/addresses?chain=eth&limit=10" \
  -H "Authorization: Bearer sk_live_abc123..."
```

## Pagination

Use `offset` and `limit` to paginate through results:

```bash
# First page
GET /addresses?limit=50&offset=0

# Second page
GET /addresses?limit=50&offset=50

# Third page
GET /addresses?limit=50&offset=100
```

The `has_more` field indicates if more results exist.

---

## Filtering Examples

```bash
# All ETH addresses
GET /addresses?chain=eth

# All addresses for a specific user
GET /addresses?external_id=user_123456

# ETH addresses for a specific user
GET /addresses?chain=eth&external_id=user_123456
```
